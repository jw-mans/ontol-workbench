# Проект ontol_v3_students: UML DSL с TDL и Graphviz

## Обзор проекта

**ontol_v3_students** — это комплексная система для работы с UML-диаграммами классов, включающая:

- **Text Diagram Language (TDL)** — текстовый язык описания диаграмм
- **Автоматическая генерация SVG** через Graphviz
- **Обратный парсинг SVG** в Pydantic-модели
- **Семантическая валидация** диаграмм
- **Веб-интерфейс** для интерактивной работы

**Дата разработки:** Апрель 2026  
**Технологии:** Python 3.10+, Pydantic v2, Graphviz, Streamlit, BeautifulSoup

---

## Цель работы

### Основная задача
Создать полнофункциональную систему для:
1. **Описания UML-диаграмм** в текстовом виде (TDL)
2. **Автоматической генерации** качественных SVG-диаграмм
3. **Валидации семантики** диаграмм
4. **Обратного парсинга** SVG в структурированные модели

### Ключевые требования
- **Детерминированная генерация** — одинаковый TDL → одинаковый SVG
- **Полная семантическая валидация** — циклы наследования, конфликты типов, ссылочная целостность
- **Обратная совместимость** — SVG → Pydantic-модели
- **Интерактивный веб-интерфейс** для работы с диаграммами

### Проблематика
- **Ручное позиционирование** элементов приводило к "кривыми стрелкам"
- **Отсутствие валидации** семантики диаграмм
- **Недетерминированная генерация** SVG
- **Отсутствие обратного парсинга** из SVG

## Архитектура системы

### Общая схема работы

```
TDL текст → Лексер → Парсер → AST → Сборка модели → Graphviz → SVG с data-атрибутами
     ↑                                                                       ↓
     └─────────────────── Streamlit веб-интерфейс ───────────────────────────┘
                                     ↓
                            Семантическая валидация
```

### Основные компоненты

#### 1. **Ядро модели (Pydantic)**
- **`models.py`** — базовые элементы UML (классы, атрибуты, операции)
- **`relationships.py`** — отношения между классами (ассоциации, зависимости, обобщения)
- **`enums.py`** — перечисления (видимость, стереотипы, типы агрегации)
- **`diagram.py`** — контейнер диаграммы и глобальная валидация

#### 2. **Парсер TDL (Text Diagram Language)**
- **`tdl_lexer.py`** — лексический анализатор
- **`tdl_parser.py`** — синтаксический анализатор
- **`tdl_ast.py`** — абстрактное синтаксическое дерево
- **`tdl_build.py`** — сборка Pydantic-модели из AST

#### 3. **Генерация SVG**
- **`graphviz_render.py`** — рендеринг через Graphviz (новая реализация)
- **`diagram.py`** — устаревший рендеринг (legacy, для обратной совместимости)

#### 4. **Парсер SVG**
- **`svg_parser.py`** — обратный парсинг SVG в Pydantic-модели

#### 5. **Веб-интерфейс**
- **`app.py`** — Streamlit-приложение с двумя вкладками

#### 6. **Утилиты**
- **`export.py`** — экспорт в PNG/JPG через CairoSVG/Pillow
- **`__init__.py`** — экспорт основных классов

## Детальное описание файлов

### models.py
**Назначение:** Базовые модели UML элементов  
**Ключевые классы:**
- `Class` — UML-класс с атрибутами и операциями
- `Attribute` — атрибут класса
- `Operation` — операция класса
- `Parameter` — параметр операции
- `MultiplicityRange` — диапазон кратности

**Особенности:**
- Полная валидация типов и значений
- Поддержка всех UML-конструкций (абстрактные классы, стереотипы)
- SVG-рендеринг отдельных элементов

### relationships.py
**Назначение:** Отношения между классификаторами  
**Ключевые классы:**
- `Association` — ассоциация с полюсами
- `AssociationEnd` — полюс ассоциации
- `Dependency` — зависимость
- `Generalization` — обобщение
- `Realization` — реализация

**Особенности:**
- Поддержка агрегации и композиции
- Стереотипы зависимостей
- N-арные ассоциации

### diagram.py
**Назначение:** Контейнер диаграммы и глобальный рендеринг  
**Ключевые классы:**
- `ClassDiagram` — основная модель диаграммы
- `ClassPosition` — позиционирование элементов

**Методы:**
- `validate_all()` — семантическая валидация
- `to_svg()` — рендеринг через Graphviz (текущий)
- `to_svg_legacy()` — старый рендеринг (для совместимости)
- `to_html()` — интерактивный HTML

### tdl_lexer.py
**Назначение:** Лексический анализатор TDL  
**Функции:**
- `lex(text)` — разбор текста на токены
- Обработка ключевых слов, идентификаторов, литералов

### tdl_parser.py
**Назначение:** Синтаксический анализатор TDL  
**Функции:**
- `parse_tdl(tokens)` — построение AST
- Обработка грамматики TDL

### tdl_ast.py
**Назначение:** Абстрактное синтаксическое дерево  
**Классы:**
- `Document` — корень AST
- `ClassDef`, `AttrDef`, `OpDef` — определения элементов
- `AssocDef`, `DepDef`, `GenDef` — определения отношений

### tdl_build.py
**Назначение:** Сборка Pydantic-модели из AST  
**Функции:**
- `build_diagram(ast)` — создание ClassDiagram
- Преобразование AST в бизнес-объекты

### tdl_run.py
**Назначение:** Точка входа для командной строки  
**Функции:**
- `tdl_to_svg(text)` — полный цикл TDL → SVG
- `main()` — обработка аргументов командной строки

### graphviz_render.py
**Назначение:** Генерация SVG через Graphviz  
**Функции:**
- `diagram_to_dot()` — преобразование в DOT-формат
- `enrich_svg_with_data_attrs()` — добавление data-атрибутов
- `diagram_to_graphviz_svg()` — полный цикл

**Особенности:**
- Автоматическая раскладка через Graphviz
- Детерминированные результаты
- Data-атрибуты для обратного парсинга

### svg_parser.py
**Назначение:** Парсер SVG обратно в Pydantic-модели  
**Функции:**
- `parse_svg_to_diagram()` — главный парсер
- `ParseResult` — результат с ошибками/предупреждениями

**Особенности:**
- Двухуровневая валидация (синтаксис + семантика)
- Сбор ошибок вместо исключений
- Поддержка всех типов отношений

### app.py
**Назначение:** Веб-интерфейс на Streamlit  
**Вкладки:**
- **"Рендер TDL"** — ввод TDL, генерация SVG
- **"Валидатор SVG"** — загрузка SVG, двухуровневая валидация

**Особенности:**
- Интерактивная работа
- Экспорт в PNG/JPG
- Детальная диагностика ошибок

## Взаимосвязи файлов

### Зависимости импортов

```
tdl_run.py
├── tdl_lexer.py (lex)
├── tdl_parser.py (parse_tdl)
├── tdl_build.py (build_diagram)
└── graphviz_render.py (diagram_to_graphviz_svg)

app.py
├── svg_parser.py (parse_svg_to_diagram)
├── diagram.py (ClassDiagram)
├── export.py (svg_to_png, svg_to_jpg)
├── tdl_run.py (tdl_to_svg)
├── tdl_lexer.py (lex)
├── tdl_parser.py (parse_tdl)
└── tdl_build.py (build_diagram)

graphviz_render.py
└── diagram.py (ClassDiagram)

svg_parser.py
├── models.py (Class, Attribute, Operation)
├── relationships.py (Association, Dependency, Generalization)
└── diagram.py (ClassDiagram, ClassPosition)

diagram.py
├── models.py (Classifier, Class)
└── relationships.py (Association, Dependency, Generalization)

tdl_build.py
├── tdl_ast.py (Document, ClassDef)
├── models.py (Class, Attribute)
├── relationships.py (Association, Dependency)
└── diagram.py (ClassDiagram)
```

### Поток данных

1. **Ввод TDL** → `tdl_run.py`
2. **Лексический анализ** → `tdl_lexer.py`
3. **Синтаксический анализ** → `tdl_parser.py`
4. **AST** → `tdl_ast.py`
5. **Сборка модели** → `tdl_build.py`
6. **Валидация** → `diagram.py`
7. **Генерация DOT** → `graphviz_render.py`
8. **Graphviz рендеринг** → системный `dot`
9. **Обогащение data-атрибутами** → `graphviz_render.py`
10. **Вывод SVG**

### Обратный поток (SVG → Модель)

1. **Ввод SVG** → `svg_parser.py`
2. **Парсинг XML** → встроенный `xml.etree.ElementTree`
3. **Извлечение data-атрибутов** → `svg_parser.py`
4. **Сборка Pydantic-модели** → `svg_parser.py`
5. **Валидация** → `diagram.py`
6. **Вывод результата** → `ParseResult`

## Запуск и использование

### Системные требования
- Python 3.10+
- Graphviz (системный)
- Зависимости: `pip install -r requirements.txt`

### Быстрый старт

```bash
# 1. Генерация SVG из TDL
python -m uml_dsl.tdl_run examples/example.tdl

# 2. Запуск веб-интерфейса
streamlit run uml_dsl/app.py

# 3. Тестирование парсера
cd examples
python -m imported_examples  # генерация тестов
python -m test_parser        # тестирование
```
