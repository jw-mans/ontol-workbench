# ONTOL V1 — грамматика языка (РБНФ)

Грамматика выведена из лексера и парсера ([src/ontol/parser.py](../src/ontol/parser.py))
и множеств допустимых значений из ([src/ontol/oast.py](../src/ontol/oast.py)).

Нотация РБНФ: `=` — определение, `;` — конец правила, `|` — альтернатива,
`[ ]` — необязательная часть, `{ }` — повторение (ноль и более раз),
`( )` — группировка, `"..."` — терминал, `? ... ?` — внешнее (лексическое) определение.

```rbnf
(* ============================ Структура программы ============================ *)

программа           = { оператор } ;

оператор            = мета_тег
                    | импорт
                    | блок_типов
                    | блок_функций
                    | блок_иерархии
                    | фигура
                    | новая_строка ;

(* ================================ Метаданные ================================ *)

мета_тег            = имя_мета, ":", строка, новая_строка ;
имя_мета            = "version" | "title" | "author"
                    | "description" | "type" | "date" ;

(* ================================== Импорт ================================== *)

импорт              = "import", ( "*" | список_импорта ), "from", строка ;

список_импорта      = "{", [ новая_строка ],
                      [ элемент_импорта,
                        { ",", [ новая_строка ], элемент_импорта },
                        [ "," ], [ новая_строка ] ],
                      "}" ;
элемент_импорта     = идентификатор, [ "as", идентификатор ] ;

(* =============================== Блок типов ================================= *)

блок_типов          = "types", ":", новая_строка, { тип, новая_строка } ;
тип                 = идентификатор, ":", строка, ",", строка, [ атрибуты ] ;

(* ============================== Блок функций =============================== *)

блок_функций        = "functions", ":", новая_строка, { функция, новая_строка } ;
функция             = идентификатор, ":", строка, параметры, "->",
                      идентификатор, ":", строка, [ атрибуты ] ;
параметры           = "(", [ новая_строка ],
                      [ параметр,
                        { ",", [ новая_строка ], параметр },
                        [ "," ], [ новая_строка ] ],
                      ")" ;
параметр            = идентификатор, ":", строка ;

(* ===================== Блок иерархии (отношения) =========================== *)

блок_иерархии       = "hierarchy", ":", новая_строка, { отношение, новая_строка } ;
отношение           = [ идентификатор, ":" ],
                      идентификатор, тип_отношения, идентификатор, [ атрибуты ] ;
тип_отношения       = "dependence" | "association" | "directAssociation"
                    | "inheritance" | "implementation"
                    | "aggregation" | "composition" ;

(* =========================== Фигуры (подсхемы) ============================= *)

фигура              = "figure", строка, ":", новая_строка,
                      { идентификатор, новая_строка } ;

(* ================================ Атрибуты ================================= *)

атрибуты            = ",", "{", [ новая_строка ],
                      [ атрибут,
                        { ",", [ новая_строка ], атрибут },
                        [ "," ], [ новая_строка ] ],
                      "}" ;
атрибут             = идентификатор, ":", строка ;

(* ================================= Лексемы ================================= *)

идентификатор       = ( буква | "_" ), { буква | цифра | "_" } ;
строка              = "'", { ? любой символ, кроме "'" ? }, "'"
                    | '"', { ? любой символ, кроме '"' ? }, '"' ;
буква               = "A" | … | "Z" | "a" | … | "z" ;
цифра               = "0" | … | "9" ;
новая_строка        = ? один или несколько символов перевода строки ? ;
```

## Семантические ограничения (вне контекстно-свободной грамматики)

Эти условия проверяются парсером, а не самой грамматикой:

| Где | Ограничение |
|-----|-------------|
| `имя_мета` | лексически — идентификатор; допустимы только перечисленные имена. |
| `тип_отношения` | лексически — идентификатор; допустимы только 7 значений выше. |
| ключи атрибутов в **типах** | `color`, `note`. |
| ключи атрибутов в **функциях** | `color`, `colorArrow`, `type`, `inputTitle`, `outputTitle`; значение `type` — один из `тип_отношения`. |
| ключи атрибутов в **отношениях** | `color`, `direction`, `title`, `rightChar`, `leftChar`; значение `direction` ∈ `forward` \| `backward` \| `bidirectional`. |
| ссылки на термы | в параметрах/функциях/отношениях/фигурах идентификатор должен ссылаться на уже определённый терм/определение. |
| уникальность имён | повторное определение имени — ошибка. |
| импорт | путь в `from '...'` — относительный (резолвится от папки файла) или URL; циклические импорты запрещены. |

## Лексические замечания

- Пробелы и табуляции между лексемами игнорируются.
- Комментарий: `#` и далее любые символы до конца строки — игнорируется.
- Строки заключаются в одинарные либо двойные кавычки; внутри `\n` трактуется как перенос (в метках).

## Пример

```ontol
version: '1.0'
title: 'Subsets'
author: ''
description: ''

import { set, element } from 'base.ontol'

types:
subset: 'Subset', '', { color: '#E6B8B7' }

hierarchy:
subset inheritance set
element aggregation set, { leftChar: '*' }
```
